# Leads Sales Checklist + Deals Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Post-qualification sales checklist on every lead (KB-driven via AI, default fallback, comment-gated task completion) + drag-and-drop deals pipeline over the existing funnel with a mandatory loss reason, surfaced compactly on /leads and /dashboard.

**Architecture:** The funnel stays the single stage machine — a new terminal `lost` stage plus server-side gates in `funnel.setStage` (won ⇐ checklist complete; lost ⇐ reason). One `salesChecklists` row per qualification session, generated by a scheduled internal action on `completeQualification` (KB `SALES CHECKLIST` sentinel → LLM strict-JSON → default 6-step fallback). Item completion requires a note and mirrors to `contactNotes` (the AI-processable trail). `leadsBoard` carries checklist + stage per lead; `/leads` gains a List|Pipeline toggle.

**Tech Stack:** Convex (self-hosted — NEVER run `convex dev`/`codegen`; hand-edit `_generated/api.d.ts`), convex-test + Vitest, Next.js App Router, shadcn UI, next-intl (single locale `messages/en.json`), native HTML5 DnD.

## Global Constraints

- Additive schema only; `npx convex deploy` BEFORE the Netlify merge (standing rule).
- New convex module = hand-edit `convex/_generated/api.d.ts` (2 lines: import + fullApi entry).
- All UI copy via `messages/en.json` (no hardcoded strings); every mutation call in try/catch with `toast.error`.
- RBAC: mirrors `funnel.setStage` — `requireRole("agent")` + `requireConversationAccess(conversationId, "own")`; viewers read-only.
- `leads-board-view.tsx` stays presentational/mock-renderable: mutations enter via props.
- Loss categories fixed: `price, competitor, budget, timing, unresponsive, changed_plans, other`.
- Default checklist keys/order: `call, pitch, price, negotiate, follow_up, objection`.
- Tests offline (`CONVEX_AI_DRY_RUN` for LLM paths); run `npx vitest run <file>`, `npx tsc --noEmit`, `npx eslint <files>`; commit after each green task with exact paths (never `git add -A`).
